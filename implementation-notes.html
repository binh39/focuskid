FocusKid ADHD/HCI redesign notes

Decisions not specified in the original request

  • Chọn hướng thiết kế Calm Adventure cho trẻ 8-12 tuổi: vui nhưng ít nhiễu, dùng màu pastel dịu và một hành động chính trên mỗi màn.
  • Đặt thời lượng tập trung mặc định là 15 phút thay vì 25 phút để phù hợp hơn với trẻ ADHD.
  • Thêm focusHelperEnabled vào preferences để camera helper có thể tắt/bật thay vì luôn tạo cảm giác bị theo dõi.
  • Đổi camera từ tự bật sang opt-in: trẻ chủ động bấm Start helper sau khi đọc giải thích quyền riêng tư.
  • Đổi popup mất tập trung từ cảnh báo đỏ sang dialog nghỉ nhẹ, dùng ngôn ngữ khuyến khích thay vì trách phạt.
  • Ưu tiên phần code có giá trị demo HCI cao nhất trong Priority 2: analytics metadata, TTS bằng trình duyệt và badge nhẹ; tạm bỏ qua quản lý nhiều tài khoản con để giữ scope ổn định.
  • Thiết kế analytics theo hướng hỗ trợ phụ huynh hiểu quá trình học, không biến camera helper thành công cụ giám sát.

Changes from the existing code

  • Thêm utility preferences có kiểm thử để lưu cài đặt tập trung vào localStorage.
  • Chuyển Settings từ UI local-only thành cài đặt có tác dụng cho timer/camera/sound reminder.
  • Thiết kế lại Child Dashboard thành màn một nhiệm vụ chính hôm nay, giảm cụm thống kê và sidebar kiểu người lớn.
  • Làm mềm Reader focus helper: camera optional, có safety copy, nút stop, và popup pause nhẹ.
  • Thêm CSS Calm Adventure với nền pastel, card bo tròn, màu mint/cream/lavender và progress semantic.

Tradeoffs

  • Dùng localStorage thay vì backend để hoàn thành nhanh trong phạm vi frontend/HCI prototype; dữ liệu đủ ổn cho demo nhưng chưa đồng bộ đa thiết bị.
  • Giữ cấu trúc file hiện tại để thay đổi tập trung, tránh refactor lớn ngoài phạm vi UI/UX.
  • Chưa làm hệ thống i18n đầy đủ; copy mới ưu tiên tiếng Anh nhất quán với phần lớn UI hiện có.
  • Chưa làm child selector/multi-child management trong giai đoạn này vì dễ mở rộng scope; parent insights vẫn dùng quan hệ parent-child hiện có ở backend.
  • Dùng browser SpeechSynthesis cho TTS để đáp ứng multimodal interaction mà không thêm backend hoặc dịch vụ ngoài.

Task 4 speech utility notes

  • Decision: kept the SpeechSynthesis helper browser-only and dependency-free so it can support multimodal read-aloud without backend or third-party services.
  • Change from plan/code: added a small utility API that returns false for unsupported or empty text so UI callers can fallback safely.
  • Tradeoff: uses the browser default voice instead of voice selection to keep scope minimal and predictable for Task 4.
  • Change log: src/utils/speech.ts adds canSpeakText, cancelSpeech, and speakText; src/utils/speech.test.ts covers unsupported browsers, speak flow, and safe cancel.

Task 5 reader read-aloud controls

  • Decision: added read-aloud entry points in both the timer card and quiz heading so children can hear the current step without searching the page.
  • Change from plan/code: reused the existing SpeechSynthesis helper from Task 4 and disabled controls when speech is unsupported or no file/quiz is active.
  • Tradeoff: the file read-aloud uses short mission guidance instead of attempting to read PDF/iframe content, keeping scope safe and browser-compatible.
  • Change log: src/pages/ChildReader.tsx adds read-aloud buttons and handlers; src/assets/reader.css adds compact pill button styles for timer and quiz controls.

Task 6 parent analytics display

  • Decision: placed focus insights in the existing parent sidebar so mission assignment remains the main page task while analytics stay visible.
  • Change from plan/code: the current parent dashboard did not yet render a focus insights card, so Task 6 added the fetch/display integration before adding the requested daily pauses and recent support moments breakdowns.
  • Tradeoff: kept summaries compact and supportive instead of adding charts, preserving the low-pressure HCI direction for ADHD support.
  • Change log: src/pages/ParentDashboard.tsx now fetches focus insights, formats daily pause dates, and renders recent sessions, daily pauses, and support moments; src/assets/dashboard.css adds spacing and row styles for the insight breakdowns.

Task 7 reward badges

  • Decision: keep badge derivation purely XP-based so the UI can celebrate progress without tracking streak failures or other punitive signals.
  • Change from plan/code: rewards.ts had rank helpers but no badge model, so Task 7 adds a lightweight badge catalog and a derived getter for earned badges.
  • Tradeoff: the badge set is intentionally small and deterministic; it covers the requested motivation moments without adding backend state or extra profile fields.
  • Change log: src/utils/rewards.ts adds RewardBadge, REWARD_BADGES, and getEarnedBadges; src/utils/rewards.test.ts verifies the starter badge always appears and higher XP unlocks calm-reader and quiz-star without any lost-streak badge.

Task 8 reward and rank badges

  • Decision: rendered the same earned badge set on both Reward and Rank pages so children see one consistent celebration model wherever they check progress.
  • Change from plan/code: reused the existing XP-based badge helper from Task 7 instead of introducing a second badge source or extra badge state.
  • Tradeoff: kept the badge cards informational only, with no interactive controls, to avoid turning reward screens into another task list.
  • Change log: src/pages/reward.tsx and src/pages/ChildRanks.tsx now import badge icons and render earned badges; src/assets/reward.css and src/assets/ranks.css add responsive badge panels and card styles.

Default mission templates

  • Decision: implemented templates as frontend-only editable presets so parents can review and adjust quiz content before creating a mission.
  • Change from plan/code: kept file uploads manual; templates only prefill the title and quiz builder because files are child/class-specific.
  • Tradeoff: static templates are fast and reliable for demo use, but they require code changes to add more templates later.
  • Change log: src/utils/missionTemplates.ts adds the default template catalog; src/components/AssignMission.tsx adds the chooser and apply behavior; src/assets/dashboard.css styles the template cards.

Camera helper and bilingual TTS tuning

  • Decision: reduced the focus helper reminder delay from 10 seconds to 3 seconds so demo/testing feedback appears quickly while still avoiding instant false alarms.
  • Decision: added lightweight Vietnamese-vs-English detection for browser TTS and chooses a matching vi or en voice when the browser provides one.
  • Tradeoff: language detection is heuristic and browser voices depend on the user's OS/browser; if no Vietnamese voice is installed, the app still sets lang="vi-VN" but the browser may fall back to a less natural voice.
  • Change log: src/hooks/useFocusDetection.ts exports 3-second defaults; src/utils/speech.ts adds language detection and voice selection; src/pages/ChildReader.tsx switches read-aloud helper copy to Vietnamese.

  • Decision: distraction detection should always be reported to the backend when it fires, even if child reminders are disabled, so the parent dashboard still receives the event stream.
  • Tradeoff: I am keeping the dashboard fix narrowly scoped to event logging for now; I am not introducing a full focus-session lifecycle unless the backend or UX proves it needs one.
